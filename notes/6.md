## Immutable and Mutable Types


### Immutable types

ints and strings are immutable.
This means that `int` and string objects are `const`.
Recall that the following code generates the picture below.

```python
a = 0
b = '0'
```

```box-drawing
  a ───────────────► 0
  b ───────────────► '0'
```

- The object `0` will never change.
- The object `'0'` will never change.

The only way that one can influence
the object referenced by a variable
of type `int` or string is by making
the variable reference a new object.

```python
a = -1
b = 'another string'
```

The immutability of ints and strings is
why our abbreviated pictures can express
ints and strings as being stored directly
by list objects, omitting the reference
to the int and string objects,
without causing much confusion.


### Mutable Types

Lists are mutable.
If `L` references a list object,
we can change the list object by using `L.append`.

Recall that the following code generates the picture below.
```python
c = [1, '1', [2]]
```

```box-drawing
                     ┌           ┐               ┌   ┐
  c ───────────────► │ 1, '1', ●───────────────► │ 2 │
                     └           ┘               └   ┘
```

Both list objects in the picture can be changed,
for example, by using the following calls to `append`:
`c[2].append(3)` and `c.append(4)`.


### Consequence for default arguments

A default argument of type `int` or string
references an object which never changes
and so no funny business can occur.

A little care is necessary
when a default argument has the type `list`.
In most applications, we will not encounter an issue,
but it is good to see a couple of weird examples
so that similar confusing situations can be avoided.




## `weird_function_1`

```python
L = []

def weird_function_1(for_printing = L):
    print(for_printing)

weird_function_1()  # []

L.append(1)
weird_function_1()  # [1]

L = [11]
weird_function_1()  # [1]
```

Each function call uses the default argument.

```box-drawing
                            │    for parameter ─────────────────────► │ │
                            │   'for_printing'   │                    └ ┘

```

