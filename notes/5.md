# PIC 16A - Notes on `5-func-scope.py`




The file `5-func-scope.py` starts
by pointing out that when a function is defined,
this introduces a new variable and a function object.
It also provides an example for why this can be useful.




However, most importantly, this file addresses
what happens when a function is called...

1. A scope is introduced.

2. Local variables are created.

   - Function parameters are always local to a function call.
   - Otherwise, a variable is **local** to a function call exactly when
     **it is assigned to at some point in the function definition.**

3. Arguments are assigned to parameters.

4. The code in the function definition is executed.

   - When a variable that is not local to the function is used,
     the global scope is searched.
   - When nothing is explicitly returned, `None` is returned.
     Otherwise, return statements end a function call, and
     if the result of the function call is assigned to a variable, the returned expression is assigned to that variable.

5. The function scope is destroyed and
   any unreferenced objects are garbage collected.




## Example 1

Consider the following code.

```python
def example_1():
    print(i)

i = 1
example_1()
print(i)
```

After 1, 2, 3, and 4 of the function call, the picture looks as follows.

```box-drawing
                                  ┌──────────┐

                                  │  object  │
                                  └──────────┘

         i ─────────────────────►  1


     ┌┄┄┄┄┄┄┄┐
     ┊       ┊
     ┊       ┊
     ┊       ┊
     ┊ scope ┊
     └┄┄┄┄┄┄┄┘
```

Output...

## Example 3

Consider the following code.

```python
def example_3():
    L.append(3)
    L = []
    print(L)

L = [0, 1, 2]
example_3()
print(L)
```

After 1, 2, and 3 of the function call, the picture looks as follows.

```box-drawing
                                  ┌──────────┐
 example_3 ─────────────────────► │ function │
                                  │  object  │
                                  └──────────┘

         L ─────────────────────►  [0, 1, 2]


     ┌┄┄┄┄┄┄┄┐
     ┊       ┊
     ┊   L   ┊
     ┊       ┊
     ┊ scope ┊
     └┄┄┄┄┄┄┄┘
```

In step 4,
`L.append(3)` does not make sense because
the local variable `L` does not have a value yet.
This is expressed by Python as an `UnboundLocalError`.

## Example 5

Consider the following code.

```python
def example_5(L_1):
    print(g_1)

    L_1.append(-11)

    L_1 = [11]
    L_2 = [22]

    return L_2

g_1 = [1]
g_2 = [2]
g_3 = example_5(g_2)

print(g_1, g_2, g_3)
```

