## A diagram involving class objects and instance objects

Consider the following code.

```python
class C:
    class_var = 0

    def __init__(self, i):
        self.instance_var = i

    def method(self, p1, p2):
        print(self.__class__.class_var)
        print(self.instance_var)
        print(p1, p2)

instance_1 = C(11)
instance_2 = C(22)
```

## A remark about these diagrams

Note that the bound methods in the above diagrams do not exist permanently.
They are only created at the moment we need them.

You can have Python prove this to you
by running the following code.

```python
class Human:
    species = 'Homo sapiens'

    def __init__(self, age, height):
        self.age = age
        self.height = height

    def get_height(self):
        return self.height

shaq = Human(54, 216)

print(shaq.get_height is shaq.get_height)
```

The output will be `False`.
This shows that two different
bound method objects are
created by the two expressions
saying `shaq.get_height`.

For these reasons,
I normally omit
drawing methods
as attributes of
instance objects.
Instead, I know that
they will be determined
by attributes of the object
referenced by `__class__`,
the undecorated ones which
reference function objects.

## An important correspondence

For any class `C` and instance `c` of `C`,
we have the following important correspondence.

| Undecorated attribute referencing a function object |   Bound method of `c`    |
|:---------------------------------------------------:|:------------------------:|
|              `C.f`                                  | `c.f`                    |
|              `C.f(c, arg_1, ..., arg_n)`            | `c.f(arg_1, ..., arg_n)` |

In my head, I always call this "the switcheroo". Something similar happens when we create an instance of a class. Suppose `c` was created by writing `c = C(arg_1, ..., arg_n)`. When this line of code executes, the following two things happen.

 1. A variable `c` is created, referencing a newly made instance object. This new instance object has a `__class__` attribute which references the same class object as `C`.
 
 2. `C.__init__(c, arg_1, ..., arg_n)` is called, and gets resolved just like any other function call we've seen. In particular, the first argument (in this case `c`) which references the newly made instance object is assigned to the first parameter of `__init__`, usually called `self`.
