## A picture for understanding variables in Python.

Definition 5.2.1.1 on page 21 of the PIC 10A Textbook (included on BruinLearn)
introduces a picture for thinking about variables and
the values that they contain in C++.

 - This picture is used
   to explain the concept of assignment
   in definition 5.2.1.3 and example 5.2.2.3.

 - It allows for
   a graphical representation
   of scope in example 8.2.1 and
   of function scope on pages 125-131, 143, and 149.

 - Finally, it enables
   a description of references
   in definition 10.2.1 and example 10.3.4.

The picture of a variable in C++ is
incredibly useful for understanding
how C++ works.

The picture for variables and values in Python is just as useful,
and it allows one to contrast the behavior of Python with that of C++.
Suppose that `var` is a variable in Python that has been given a value
in some way. Then the following is a good mental image to have.

```box-drawing
    ┌───────────┐
    │   type    │
var │           │
    │  obj ref ─┼───────────► object
    └───────────┘
```


The value stored by a variable in Python,
what is inside the box labeled by the variable,
consists of two things:

- a type;
- a reference to an object.


To illustrate, we provide three examples.
Suppose the following Python code is executed.

```python
a = 0
b = '0'
c = [1, '1', [2]]
```

Then a good mental picture is as follows.

```box-drawing
    ┌───────────┐
    │    int    │
  a │           │
    │  obj ref ─┼───────────► 0
    └───────────┘

    ┌───────────┐
    │    str    │
  b │           │
    │  obj ref ─┼───────────► '0'
    └───────────┘

    ┌───────────┐
    │   list    │             ┌ ┌───────────┐
  c │           │             │ │    int    │
    │  obj ref ─┼───────────► │ │           │
    └───────────┘             │ │  obj ref ─┼───────────► 1
                              └ └───────────┘,
                                ┌───────────┐
                                │    str    │
                                │           │
                                │  obj ref ─┼───────────► '1'
                                └───────────┘,
                                ┌───────────┐ ┐
                                │   list    │ │           ┌ ┌───────────┐ ┐
                                │           │ │           │ │    int    │ │
                                │  obj ref ─┼───────────► │ │           │ │
                                └───────────┘ ┘           │ │  obj ref ─┼───────────► 2
                                                          └ └───────────┘ ┘
```

Rather than writing `obj ref` for "object reference" everywhere,
we will use a circle for the base of an object reference.

```box-drawing
    ┌───────┐
    │  int  │
  a │       │
    │   ●───┼───────────► 0
    └───────┘

    ┌───────┐
    │  str  │
  b │       │
    │   ●───┼───────────► '0'
    └───────┘

    ┌───────┐
    │ list  │             ┌ ┌───────┐
  c │       │             │ │  int  │
    │   ●───┼───────────► │ │       │
    └───────┘             │ │   ●───┼───────────► 1
                          └ └───────┘,
                            ┌───────┐
                            │  str  │
                            │       │
                            │   ●───┼───────────► '1'
                            └───────┘,
                            ┌───────┐ ┐
                            │ list  │ │           ┌ ┌───────┐ ┐
                            │       │ │           │ │  int  │ │
                            │   ●───┼───────────► │ │       │ │
                            └───────┘ ┘           │ │   ●───┼───────────► 2
                                                  └ └───────┘ ┘
```

The picture for the variable `c` is probably
much more complicated than you expected.
However, a list contains values.
Each of these values consists of
a type and an object reference.

In many cases, I would abbreviate the picture above to the following one,
but for the examples of `4-variables.py`, I will first use completely
unabbreviated pictures. Abbreviated pictures can be found


## Example 1

Consider the following code.

```python
L1 = [0]
L2 = L1

L1.append(1)
L2.append(2)

print(L1, L2)
```

We understand the first line:
`L1 = [0]` creates the following picture.

```box-drawing
    ┌───────┐
    │ list  │             ┌ ┌───────┐ ┐
 L1 │       │             │ │  int  │ │
    │   ●───┼───────────► │ │       │ │
    └───────┘             │ │   ●───┼───────────► 0
                          └ └───────┘ ┘
```

Onto the second line: `L2 = L1`.
 - In C++, if we have two ints `i` and `j`,
   the assignment `i = j` updates `i`'s value using `j`'s value.
   This occurs in Example 5.2.2.3 of the PIC 10A textbook.
 - The same happens in Python when we write `L2 = L1`:
   `L2`'s value is updated using `L1`'s value.
 - **However,** we just learned that "value" consists
   of two things in Python: a type and an object reference.
 - `L2 = L1` ensures that `L2` has the same value as `L1`.
   Therefore, `L2` also records the type `list` and
   it stores an object reference to the same object
   as `L1`'s object reference.

```box-drawing
    ┌───────┐
    │ list  │             ┌ ┌───────┐ ┐
 L1 │       │             │ │  int  │ │
    │   ●───┼───────────► │ │       │ │
    └───────┘     ╭─────► │ │   ●───┼───────────► 0
                  │       └ └───────┘ ┘
    ┌───────┐     │
    │ list  │     │
 L2 │       │     │
    │   ●───┼─────╯
    └───────┘
```

Therefore, `L1.append(1)` and `L2.append(2)`
append to the same list object...

```box-drawing
    ┌───────┐
    │ list  │             ┌ ┌───────┐
 L1 │       │             │ │  int  │
    │   ●───┼───────────► │ │       │
    └───────┘     ╭─────► │ │   ●───┼───────────► 0
                  │       └ └───────┘,
    ┌───────┐     │         ┌───────┐
    │ list  │     │         │  int  │
 L2 │       │     │         │       │
    │   ●───┼─────╯         │   ●───┼───────────► 1
    └───────┘               └───────┘,
                            ┌───────┐ ┐
                            │  int  │ │
                            │       │ │
                            │   ●───┼───────────► 2
                            └───────┘ ┘
```

...and when we print `L1` and `L2`,
we see the list `[0, 1, 2]` twice.




## Example 2

Consider the following code.

```python
L1 = [0]
L2 = L1

L1 = L1 + [1]
L2 = L2 + [2]

print(L1, L2)
```

We understand the first two lines
because they are the same as the previous example.

```box-drawing
    ┌───────┐
    │ list  │
 L1 │       │
    │   ●───┼─────╮
    └───────┘     │
                  │
                  │       ┌ ┌───────┐ ┐
                  │       │ │  int  │ │
                  ╰─────► │ │       │ │
                  ╭─────► │ │   ●───┼───────────► 0
                  │       └ └───────┘ ┘
    ┌───────┐     │
    │ list  │     │
 L2 │       │     │
    │   ●───┼─────╯
    └───────┘
```

Onto the third line: `L1 = L1 + [1]`.
 - In C++, if we have three ints `i`, `j`, and `k`.
   the assignment `i = j + k` updates `i`'s value
   by evaluating `j + k`. This occurs in
   Example 5.2.2.3 of the PIC 10A textbook.
 - The same happens in Python when we write `L1 = L1 + [1]`:
   `L1`'s value is updated by evaluating `L1 + [1]`.
 - The expression `L1 + [1]` creates a new object
   together with a value which stores a reference to it.
   This is shown to the right of the picture below.
   ```box-drawing
       ┌───────┐                                     ┌───────┐
       │ list  │                                     │ list  │             ┌ ┌───────┐
    L1 │       │                                     │       │             │ │  int  │
       │   ●───┼─────╮                               │   ●───┼───────────► │ │       │
       └───────┘     │                               └───────┘             │ │   ●───┼───────────► 0
                     │                                                     └ └───────┘,
                     │       ┌ ┌───────┐ ┐                                   ┌───────┐ ┐
                     │       │ │  int  │ │                                   │  int  │ │
                     ╰─────► │ │       │ │                                   │       │ │
                     ╭─────► │ │   ●───┼───────────► 0                       │   ●───┼───────────► 1
                     │       └ └───────┘ ┘                                   └───────┘ ┘
       ┌───────┐     │
       │ list  │     │
    L2 │       │     │
       │   ●───┼─────╯
       └───────┘
   ```
 - Finally, this value is used to overwrite `L1`'s value.
   ```box-drawing
       ┌───────┐
       │ list  │                                                           ┌ ┌───────┐
    L1 │       │                                                           │ │  int  │
       │   ●───┼─────────────────────────────────────────────────────────► │ │       │
       └───────┘                                                           │ │   ●───┼───────────► 0
                                                                           └ └───────┘,
                             ┌ ┌───────┐ ┐                                   ┌───────┐ ┐
                             │ │  int  │ │                                   │  int  │ │
                             │ │       │ │                                   │       │ │
                     ╭─────► │ │   ●───┼───────────► 0                       │   ●───┼───────────► 1
                     │       └ └───────┘ ┘                                   └───────┘ ┘
       ┌───────┐     │
       │ list  │     │
    L2 │       │     │
       │   ●───┼─────╯
       └───────┘
   ```

Similarly, the fourth line, leads to the creation of a new object
which is referenced by `L2`'s value.

```box-drawing
    ┌───────┐
    │ list  │                                                           ┌ ┌───────┐
 L1 │       │                                                           │ │  int  │
    │   ●───┼─────────────────────────────────────────────────────────► │ │       │
    └───────┘                                                           │ │   ●───┼───────────► 0
                                                                        └ └───────┘,
                          ┌ ┌───────┐ ┐                                   ┌───────┐ ┐
                          │ │  int  │ │                                   │  int  │ │
                          │ │       │ │                                   │       │ │
                          │ │   ●───┼───────────► 0                       │   ●───┼───────────► 1
                          └ └───────┘ ┘                                   └───────┘ ┘
    ┌───────┐
    │ list  │                                                           ┌ ┌───────┐
 L2 │       │                                                           │ │  int  │
    │   ●───┼─────────────────────────────────────────────────────────► │ │       │
    └───────┘                                                           │ │   ●───┼───────────► 0
                                                                        └ └───────┘,
                                                                          ┌───────┐ ┐
                                                                          │  int  │ │
                                                                          │       │ │
                                                                          │   ●───┼───────────► 2
                                                                          └───────┘ ┘
```

Because the list object of length 1 is not referenced anymore,
Python's garbage collection gets rid of it.

```box-drawing
    ┌───────┐
    │ list  │                                                           ┌ ┌───────┐
 L1 │       │                                                           │ │  int  │
    │   ●───┼─────────────────────────────────────────────────────────► │ │       │
    └───────┘                                                           │ │   ●───┼───────────► 0
                                                                        └ └───────┘,
                                                                          ┌───────┐ ┐
                                                                          │  int  │ │
                                                                          │       │ │
                                                                          │   ●───┼───────────► 1
                                                                          └───────┘ ┘
    ┌───────┐
    │ list  │                                                           ┌ ┌───────┐
 L2 │       │                                                           │ │  int  │
    │   ●───┼─────────────────────────────────────────────────────────► │ │       │
    └───────┘                                                           │ │   ●───┼───────────► 0
                                                                        └ └───────┘,
                                                                          ┌───────┐ ┐
                                                                          │  int  │ │
                                                                          │       │ │
                                                                          │   ●───┼───────────► 2
                                                                          └───────┘ ┘
```

When we print `L1` and `L2`,
we see the lists `[0, 1]` and `[0, 2]`.

This example illustrates a very important difference between C++ and Python: **reassigning a variable in Python will cause a new object to be created, and the object reference stored by that variable (i.e. the arrow) will be moved to point at the newly created object.** If you naively follow the arrow and modify what's at the end of it, you'll end up with behaviour that's more closely aligned with C++.

## Example 4

Consider the following code.

```python
L1 = [8 * '8']
L2 = [8 * '8']

print(L1 == L2)
print(L1 is L2)
```

The second line `L2 = [8 * '8']` is very different from `L2 = L1`.
This reveals that we did not think as deeply as we might have done
when seeing the first line `L1 = [8 * '8']`.

 - The expression `[8 * '8']` creates a new list object
   together with a value which stores a reference to it;
   `L1` stores this value.

The line `L2 = [8 * '8']` does basically
the same thing (with `L2` replacing `L1`).

 - The expression `[8 * '8']` creates a new list object
   together with a value which stores a reference to it;
   `L2` stores this value.

We are left with the following picture.

```box-drawing
    ┌───────┐
    │ list  │             ┌ ┌───────┐ ┐
 L1 │       │             │ │  str  │ │
    │   ●───┼───────────► │ │       │ │
    └───────┘             │ │   ●───┼───────────► '88888888'
                          └ └───────┘ ┘
    ┌───────┐
    │ list  │             ┌ ┌───────┐ ┐
 L2 │       │             │ │  str  │ │
    │   ●───┼───────────► │ │       │ │
    └───────┘             │ │   ●───┼───────────► '88888888'
                          └ └───────┘ ┘
```

- When we print `L1 == L2`, we get `True` because `len(L1) == len(L2)` and `L1[0] == L2[0]`.
- When we print `L1 is L2`, we get `False` because `L1` and `L2` reference different list objects.

## Example 5

Consider the following code.

```python
def trial(L):
    L.append(0)
    L = [1]

starts_empty = []

trial(starts_empty)
print(starts_empty)
```

We have still not discussed function scope in Python,
but you should always expect that any coding language
will introduce a function scope that stores the parameters.

When the function `trial` is called,
the argument-to-parameter assignment
is `L = starts_empty`.
This copies `starts_empty`'s value to `L`.
Therefore, `L` also records the type `list` and
it stores an object reference to the same object
as `starts_empty`'s object reference.

```box-drawing
              ┌───────┐
              │ list  │                    ┌  ┐
 starts_empty │       │                    │  │
              │   ●───┼──────────────────► │  │
              └───────┘              ╭───► │  │
                                     │     └  ┘
                                     │
        ┌┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┐        │
        ┊                   ┊        │
        ┊     ┌───────┐     ┊        │
        ┊     │ list  │     ┊        │
        ┊   L │       │     ┊        │
        ┊     │   ●───┼──────────────╯
        ┊     └───────┘     ┊
        ┊                   ┊
        ┊       scope       ┊
        └┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┘
```

In the function definition,
the line `L.append(0)` causes
`0` to be appended to the only
list object that exists.

```box-drawing
              ┌───────┐
              │ list  │                    ┌ ┌───────┐ ┐
 starts_empty │       │                    │ │  int  │ │
              │   ●───┼──────────────────► │ │       │ │
              └───────┘              ╭───► │ │   ●───┼───────────► 0
                                     │     └ └───────┘ ┘
                                     │
        ┌┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┐        │
        ┊                   ┊        │
        ┊     ┌───────┐     ┊        │
        ┊     │ list  │     ┊        │
        ┊   L │       │     ┊        │
        ┊     │   ●───┼──────────────╯
        ┊     └───────┘     ┊
        ┊                   ┊
        ┊       scope       ┊
        └┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┘
```

In the function definition,
the line `L = [1]` causes
`L`'s value to be updated
so that it references the
object created by the
expression `[1]`.

```box-drawing
              ┌───────┐
              │ list  │                    ┌ ┌───────┐ ┐
 starts_empty │       │                    │ │  int  │ │
              │   ●───┼──────────────────► │ │       │ │
              └───────┘                    │ │   ●───┼───────────► 0
                                           └ └───────┘ ┘

        ┌┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┐
        ┊                   ┊
        ┊     ┌───────┐     ┊
        ┊     │ list  │     ┊              ┌ ┌───────┐ ┐
        ┊   L │       │     ┊              │ │  int  │ │
        ┊     │   ●───┼──────────────────► │ │       │ │
        ┊     └───────┘     ┊              │ │   ●───┼───────────► 1
        ┊                   ┊              └ └───────┘ ┘
        ┊       scope       ┊
        └┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┄┘
```

When the function call ends,
the function scope is destroyed.
The parameter `L` is cleaned up,
and because the list object created by `[1]`
is no longer referenced, it is garbage collected.

```box-drawing
              ┌───────┐
              │ list  │                    ┌ ┌───────┐ ┐

              └───────┘                    │ │   ●───┼───────────► 0
                                           └ └───────┘ ┘
```

When we print `starts_empty`, we see `[0]`.
